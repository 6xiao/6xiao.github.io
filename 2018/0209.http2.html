HTTP2 协议很像五年前的 Golang

据说很多爱文艺的青年，其实都少有文化。

比如：我这么一个挺喜欢文艺的一个人，却漂在技术发展的浪潮上浪费时间，对面文化束手无力。

最近研究了一下 HTTP2，感觉是非常理想的适应各种场景的协议，非常像五年前的 Golang。

相似点一：并发。Golang 和 HTTP2 用更轻量的并发机制，更大规模的适应多连接场景。Golang 用比 Thread 开销小数倍的 goroutine，而 HTTP2 使用比多 TCP 更轻量的 MUX 链路复用。

相似点二：事件机制。 Golang 有 channel 替换了 Thread 轮询机制，HTTP2 可以用 Push 替换 Ajax 机制。实时性高数倍的同时，开销降了数倍。

相似点三：兼容性。HTTP2 兼容 HTTP1 ，并通过 upgrade 来切换。 Golang 做 Cgo 机制的初衷也是如此。

相似点四：非浸入式对象设计。对此我无法一句话说明白，需要你熟练掌握两者并有一定悟性。

相似点五：我喜欢。

当然它也有不足之处，最大的问题就在于其仍然以 TCP 为基础。理想状态下应该是一个新的基于 IP 层的协议，加入 FEC 和 MUX 等特性，就像 QUIC 把 UDP 换成 IP 包而又提供了完整的 HTTP2 设计，但估计这样的设计无法面对存量市场。

无论如何，这东西值得学习。为了在今年弄明白它，没文化就没文化吧。
